# Customer Churn Analysis Using MySQL

## Project Overview

This project focuses on analyzing customer churn and inactive customers in the banking sector using MySQL.

The objective is to identify:

- High churn branches
- High churn customer segments
- Product-wise churn
- Channel-wise churn
- Revenue impact of churn
- Customer complaints impact
- Branch performance

## Tools Used

- MySQL
- SQL

## Dataset Information

The dataset contains 38,360 banking records with the following attributes:

- Date
- Branch
- Product Type
- Channel
- Customer Segment
- New Accounts
- Closures
- Transactions
- Revenue
- Average Balance
- Customer Complaints
- Cost per Account
- Total Cost
- NPA Percentage
- Churn Rate

## Project Objectives

- Analyze customer churn patterns
- Identify inactive customers through account closures
- Evaluate branch performance
- Perform customer segmentation analysis
- Analyze product-wise and channel-wise churn
- Generate business reports for decision making

## Key Findings

- Average Churn Rate: 13.59%
- Net Customer Growth: 9,239
- Highest Churn Branch: Tambaram
- Highest Churn Segment: Retail
- Highest Churn Product: Loan
- Highest Churn Channel: Internet Banking
